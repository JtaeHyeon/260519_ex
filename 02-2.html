HTML5 시맨틱 웹 표준

화면에 보이지 않는 '의미(Semantic)'를 부여하여 브라우저와 개발자 모두에게 명확한 소통의 가치를 전합니다.

# 왜 시맨틱 마크업을 써야 할까요?

단순히 <div>와 <span> 태그만으로 레이아웃을 표현할 수도 있습니다. 하지만 웹 표준과 시맨틱 태그를 적절히 사용하는 것은 다음과 같은 강력한 이점을 지닙니다.

🔍

검색엔진 최적화 (SEO)

검색엔진 로봇(예: Googlebot)이 페이지 구조를 정확히 분석하여 어떤 부분이 핵심 제목이고, 어떤 부분이 내비게이션 영역인지 파악할 수 있도록 돕습니다.

♿

웹 접근성 (Accessibility)

시각 장애가 있는 사용자가 스크린 리더 기기를 사용할 때, 사이트 레이아웃의 구조를 소리로 명확히 인지하고 탐색할 수 있도록 돕는 필수적인 요소입니다.

💻

가독성 및 유지보수

수백 개의 <div>가 겹쳐진 코드보다 <header>, <main>, <aside> 등으로 구조화된 코드는 동료 개발자가 분석하기 매우 용이합니다.

# 주요 시맨틱 태그 도감

궁금한 태그를 클릭하여 해당 요소의 목적과 실제 코드 적용 방식을 확인해 보세요.

🏷️ <header>
머리말 영역을 정의합니다. 대개 로고, 사이트 이름, 대표적인 전역 내비게이션, 검색 폼 등이 여기에 포함됩니다. 페이지 전체의 상단 뿐만 아니라, <article>이나 <section>의 도입 부분에도 각각 독립적으로 삽입할 수 있습니다. 
<header>
  <h1>내 서비스 로고</h1>
  <nav>...</nav>
</header>

💡 한줄 요약

  • 시맨틱은 '의미'를 의미합니다.
  • 단순한 컨테이너(div) 대신 각 영역의 성격에 적합한 태그를 사용합시다.
  • 웹 페이지 내에 <main>은 단 하나만 존재해야 합니다.

⚖️ 태그 사용 비교

비권장 방식 (의미 무시) <div id="header">

검색 봇에게 의미가 없으며 스타일링 용도로만 동작합니다.

권장 방식 (시맨틱) <header>

기기가 직접 머리말임을 인지하고 읽습니다.
